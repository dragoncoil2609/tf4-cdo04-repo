1. Tổng Quan Phiên Làm Việc

1.1 Bối cảnh dự án

TF4 Foresight Lens giải quyết bài toán cảnh báo sớm (early warning) cho Head of SRE tại một fintech mid-size vận hành khoảng 120 microservice. Trong 3 tháng gần đây, hệ thống miss SLO 7 lần do capacity exhaustion âm thầm: RDS CPU tăng dần, SQS backlog tăng, ALB connection chạm giới hạn trong traffic spike.

Client đã có Grafana, CloudWatch và Datadog trial — vấn đề không phải thiếu dashboard hay metric. Pain chính: SRE không có workflow cảnh báo sớm đủ tin cậy, thiếu recommendation cụ thể để hành động kịp thời.

1.2 Angle của nhóm CDO-04

SLO Early-Warning Control Plane with TSDB-backed Prediction Workflow
Platform biến telemetry + AI prediction thành workflow vận hành có recommendation, evidence, audit log và fallback khi AI endpoint unavailable. Không build thêm một dashboard mới.

2.3 Luồng dữ liệu (Data Flow)

  1. Telemetry Ingestion: 3 mock service (payment-gateway, ledger-service, kyc-worker) + k6 synthetic → HTTPS → Public ALB /v1/ingest → ECS Telemetry API validate → ADOT Collector remote_write (SigV4) → AMP workspace.
  2. Prediction Orchestration: EventBridge Scheduler (mỗi 5 phút) → SQS prediction-jobs → ECS Prediction Worker poll message.
  3. Prediction Execution: Worker → query AMP bằng PromQL query_range (120 phút, filter tenant/service/metric) → align 1-minute buckets → gọi AI POST /v1/predict qua ECS Service Connect → nhận response (anomaly, severity, reasoning, recommendation.*).
  4. Audit & Alert: Worker → ghi DynamoDB audit record → publish SNS nếu high risk → S3 evidence snapshot nếu cần → delete SQS message.
  5. Fallback Path: Nếu AI timeout/unavailable/sai schema → static threshold fallback → audit ghi prediction_source=static_threshold_fallback.

3.6 Mock E2E App (src/mock-e2e/app.py)

Single Python image chạy 3 role qua MOCK_ROLE environment variable:

  • api: POST /v1/ingest → validate JSON → gửi message vào SQS prediction queue.
  • worker: Poll SQS → gọi AI /v1/predict → ghi DynamoDB audit → ghi S3 evidence → delete SQS message.
  • ai: POST /v1/predict → trả fake prediction JSON với risk_score, risk_level, flags.

Mỗi role expose GET /health trả {"status":"ok","role":"<role>"}. Sử dụng thư viện: boto3, requests, Python stdlib http.server.

5.4 Worker Flow (pseudo)

1. sqs.receive_message(WaitTimeSeconds=20, MaxNumberOfMessages=1)
2. PromQL query_range(start, end, step=60s) từ AMP → 120-min window
3. Align 1-minute buckets; forward-fill/zero-fill missing
4. Nếu window <120 phút hoặc imputation vượt ngưỡng → static fallback
5. POST ai-engine:8080/v1/predict (SigV4)
6. Nếu timeout/5xx/invalid schema → static threshold fallback
7. dynamodb.put_item(audit_record)
8. Nếu severity==high → sns.publish(alert)
9. sqs.delete_message()
10. Error recovery: retry 5 lần → DLQ

Trade-off 4: Dashboard-only vs Control Plane

Dashboard-only rẻ hơn tuyệt đối, nhưng không giải quyết đúng pain point của client: không có người nhìn dashboard 24/7, static threshold dễ quá nhạy hoặc quá trễ. Control plane phức tạp hơn (cần scheduler, queue, worker, TSDB, audit DB, alert path) nhưng đáp ứng trực tiếp lead time ≥15 phút, recommendation cụ thể, audit every prediction call, fallback khi AI unavailable.

Trade-off 5: ARM64/Graviton vs x86

ARM64 có thể giảm compute cost nhưng giữ x86 làm accepted decision cho compatibility và contract stability. ARM64 là future optimization.

8.5 Anti-Patterns (Cấm Dùng)

  • Action: "*" + Resource: "*" trong IAM policy
  • AdministratorAccess managed policy
  • Long-lived AWS access key trong container
  • Hardcode credential/secret/token trong code
  • High-cardinality Prometheus labels (request_id, trace_id, prediction_id, user_id)
  • PII trong metric, audit key hoặc Prometheus labels
  • Public ALB 0.0.0.0/0 trong non-sandbox environments

10.7 Pipeline Security Features

11. Tổng Kết

11.1 Điểm Mạnh Của Thiết Kế

  1. Fit hard budget $200/tháng với full-month x86 design + 20% buffer ($189.79/tháng).

  6. Production-like deployment: Terraform + GitHub Actions OIDC + ECS rolling deployment circuit breaker.
  7. Cost guard 50/80/100: AWS Budget alarm + rule "không tắt audit/fallback".
